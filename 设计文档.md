## 1. 项目概述

**Stellar 星辰画廊**是一个暗色沉浸式在线图片展示画廊。核心理念是将每张图片视作一颗"星辰"——在深邃暗空间中静默漂浮，用户滚动浏览时图片逐渐从灰度"睡眠态"唤醒至全彩"觉醒态"。

| 属性 | 值 |
|---|---|
| 项目名称 | Stellar — 星辰画廊 |
| 类型 | 单页面图片画廊 |
| 风格基调 | 深空暗色 · 暖金点缀 · 灰度唤醒 · 太空漂浮 |
| 技术栈 | HTML5 + CSS3 + Vanilla JavaScript |
| 字体 | Google Fonts Inter (400/500/600/700) |
| 图片源 | picsum.photos (Lorem Picsum) |

## 2. 设计哲学与视觉语言

### 2.1 核心设计理念

界面退为背景，图片成为主角。设计围绕三个关键词展开：

| 关键词 | 表达方式 |
|---|---|
| **Stellar（星辰）** | 图片如天体悬浮于暗空间，背景粒子模拟星尘 |
| **Awakening（唤醒）** | 图片从灰度(grayscale 100%)过渡至全彩，赋予生命力 |
| **Floating（漂浮）** | Canvas 粒子缓慢移动 + 鼠标吸引，营造无重力空间纵深 |

### 2.2 视觉基调

- **暗色沉浸** — 全页面使用深空黑 `#08080C` 作为底色，降低视觉噪音
- **温暖金色微光** — 金色 `#D4A853` 是唯一强调色，如星辰之光
- **灰度→彩色过渡** — 每张图片进入视口时从黑白苏醒为全彩
- **无传统阴影** — 深度由发光边框(glow border)和粒子层提供，而非 box-shadow

### 2.3 设计参考

融合三种设计语言：
- **Apple** 的摄影优先排版（极简文字 + 大图展示）
- **SpaceX** 的深空沉浸感（暗黑背景 + 粒子漂浮）
- **Airbnb** 的柔和圆角系统（12px 卡片圆角 + 9999px 胶囊按钮）

## 4. 色彩体系

### 4.1 色彩角色总览

```
深空黑画布  ──────────────────────────────────  暖金色点缀
#08080C                                          #D4A853 (主)
#111118                                          #F0D48A (亮)
#181820                                          #E5C16A (hover)
#252530
#3C3C48
```

### 4.2 主色调 — 暖金

| 变量 | 色值 | 用途 |
|---|---|---|
| `--color-accent` | `#D4A853` | 导航 Logo、悬停边框、标题、激活态滤镜按钮 |
| `--color-accent-glow` | `#F0D48A` | 发光边框高亮、粒子连线 |
| `--color-interactive-hover` | `#E5C16A` | 按钮悬停态背景 |

### 4.3 深色画布层级

| 层级 | 变量 | 色值 | 元素 |
|---|---|---|---|
| 0 — 最深 | `--color-bg-deep` | `#08080C` | 页面底色、Footer |
| 1 — 表面 | `--color-bg-surface` | `#111118` | Gallery 区域 |
| 2 — 卡片 | `--color-bg-card` | `#181820` | 图片卡片、Lightbox 工具栏、Toast |
| 3 — 遮罩 | `--color-bg-overlay` | `#0C0C14` | Lightbox 全屏遮罩 |
| 4 — 导航 | `rgba(8,8,12,0.75)` | 半透明 | 顶部导航（毛玻璃） |

### 4.4 灰度文字层级

| 色值 | 用途 | 对比度 (vs #08080C) |
|---|---|---|
| `#F0F0F5` | 主文字 | 17.8:1 (AAA) |
| `#C8C8D0` | 导航链接、活跃图标 | 10.5:1 (AAA) |
| `#9898A6` | 图片元数据（日期/地点） | 5.8:1 (AA) |
| `#686878` | Footer 文字、Lightbox 提示 | 3.1:1 (AA Large) |

### 4.5 设计禁忌

- ❌ 不使用蓝色/绿色/紫色等冷色系作为强调色
- ❌ 不使用纯白 `#FFFFFF` 作为大段正文（眩光）
- ❌ 不在图片上叠加暗色蒙版
- ✅ 金色是唯一的情感温度来源
- ✅ 正文使用 `#F0F0F5` 降低对比度至舒适范围

### 5.3 排版规则

- **展示级标题（≥36px）** 使用负字间距 `letter-spacing: -0.03em`，营造精密质感
- **正文行高 ≥1.5**，确保深色背景下长文本可读性
- **字重阶梯**: 400 → 500 → 600 → 700，排除 300（深色背景下对比度不足）
- **图片标题使用金色** `#D4A853`，形成稳定的视觉锚点

## 6. 间距系统

基于 4px 基准的 8 级间距系统：

| Token | px | rem | 典型应用 |
|---|---|---|---|
| `--space-xs` | 4px | 0.25rem | 图标与文字间微间距 |
| `--space-sm` | 8px | 0.5rem | 卡片内紧凑间距、滤镜按钮间隙 |
| `--space-md` | 16px | 1rem | 标准内边距 |
| `--space-lg` | 24px | 1.5rem | 瀑布流列间距、导航链接间隙 |
| `--space-xl` | 32px | 2rem | 导航水平内边距、区块内边距 |
| `--space-2xl` | 48px | 3rem | Footer 内边距、Gallery 标题下间距 |
| `--space-3xl` | 64px | 4rem | (预留) 大区块间隔 |
| `--space-section` | 96px | 6rem | Gallery 底部留白 |

### 留白哲学

- 图片之间保留大量"深空"间隙，每张图片独立呼吸
- 图片信息区使用渐变叠加而非框定边界
- 移动端减小水平 padding，图片占比最大化

**Glow Border 技术细节:**

`.glow-border` 伪元素使用 `radial-gradient` 跟随鼠标位置渲染金色光晕：
```css
background: radial-gradient(
  circle 200px at var(--mx) var(--my),
  rgba(212, 168, 83, 0.6),
  rgba(212, 168, 83, 0.15) 40%,
  transparent 70%
);
```
位置变量 `--mx` / `--my` 通过 JS `mousemove` 事件实时更新。

**3D 倾斜效果:**

鼠标在卡片上的相对位置映射为 `rotateX(-8deg ~ +8deg)` / `rotateY(-8deg ~ +8deg)`，使用 `perspective(1200px)` 创建空间纵深感。

## 8. 布局与网格

### 8.1 瀑布流布局

Gallery 使用 **CSS columns** 实现 Masonry 瀑布流：

| 属性 | 桌面 (≥1024px) | 平板/手机 (<1024px) |
|---|---|---|
| `column-count` | 2 | 1 |
| `column-gap` | 24px | 24px |
| 容器最大宽度 | 1400px | 100% |
| 水平内边距 | 32px | 16px (手机) |

每张卡片设置 `break-inside: avoid` 防止跨列断裂，`display: inline-block; width: 100%` 确保在各列中正确填充。

### 8.2 页面结构

### 9.2 深度媒介

本设计系统的深度**不依赖**传统 box-shadow，而是通过三种媒介表达：

1. **粒子层** — 背景浮动粒子暗示无限深空
2. **发光边框** — 悬停时金色 radial-gradient 光晕使图片"浮出"
3. **毛玻璃** — backdrop-filter 制造"悬浮于内容之上"的层次感

## 10. 动画与交互设计

### 10.1 灰度唤醒动画（核心动画）

每张图片卡片进入视口时触发"觉醒"序列：

| 阶段 | 属性 | 起始值 | 结束值 | 时长 | 缓动 |
|---|---|---|---|---|---|
| 1 — 浮入 | opacity | 0 | 1 | 800ms | cubic-bezier(0.4,0,0.2,1) |
| 1 — 浮入 | translateY | 30px | 0 | 800ms | cubic-bezier(0.4,0,0.2,1) |
| 2 — 唤醒 | filter | grayscale(100%) | grayscale(0%) | 1200ms | cubic-bezier(0.4,0,0.2,1) |

**触发条件**: IntersectionObserver，threshold 0.3（卡片 30% 可见时触发）  
**移动端优化**: filter 过渡缩短至 800ms（滚动更快，防止动画堆积）

### 10.2 3D 倾斜效果

鼠标在卡片上的相对位置映射为 3D 旋转变换：
- X 轴旋转: `-8deg ~ +8deg`（鼠标上下位置）
- Y 轴旋转: `-8deg ~ +8deg`（鼠标左右位置）
- 透视距离: `1200px`
- 鼠标离开时平滑复位

同时驱动 glow-border 的径向渐变中心点跟随鼠标。

### 10.3 粒子系统

| 参数 | 桌面 (≥1024px) | 平板 (640–1023px) | 手机 (≤639px) |
|---|---|---|---|
| 粒子数量 | 80 | 65 | 45 |
| 粒径 | 2–5px | 2–5px | 2–5px |
| 透明度 | 0.35–0.75 | 0.35–0.75 | 0.35–0.75 |
| 速度 | ±0.3 px/frame | ±0.3 px/frame | ±0.3 px/frame |
| 连线距离 | 150px | 150px | 120px |
| 鼠标影响半径 | 200px | 200px | 200px |
| 鼠标吸引力 | 0.04 | 0.04 | 0.04 |

**视觉效果:**
- 每个粒子有径向渐变发光（核心亮白，边缘透明）
- 粒子间在距离阈值内绘制金色半透明连线
- 粒子有独立呼吸脉动效果（opacity 正弦波）
- 鼠标靠近时粒子被微弱吸引
- 支持触摸交互（touchmove/touchstart/touchend）
- 窗口 resize 时重新计算粒子数量和分布

### 10.4 其他动画

| 动画 | 时长 | 缓动 |
|---|---|---|
| Lightbox 入场 | 300ms | cubic-bezier(0.4,0,0.2,1) |
| Lightbox 图片切换 | 150ms 淡出 → 切换 → 淡入 | ease |
| Lightbox 缩放提示 | 500ms 延迟显示 / 4s 后消失 | ease |
| Toast 入场 | 300ms | cubic-bezier(0.4,0,0.2,1) |
| Toast 出场 | 300ms | ease |
| 长按菜单弹出 | 200ms | cubic-bezier(0.4,0,0.2,1) |
| Shimmer 加载 | 1.5s 循环 | — |
| 滚动进度条 | 100ms 节流 | linear |

### 10.5 统一缓动函数

所有动画使用同一缓动曲线确保节奏一致：
```css
--ease-stellar: cubic-bezier(0.4, 0, 0.2, 1);
```
等价于 Material Design 标准缓动（快出慢入）。

## 附录 C: 设计决策记录

| 决策 | 原因 |
|---|---|
| 使用 CSS columns 而非 CSS Grid | 实现真正的瀑布流效果，图片高度不一时自动填充 |
| 2 列而非 3 列布局 | 在大屏上每张图片获得更大的视觉面积，更符合"沉浸式"定位 |
| 无独立 Hero 区域 | 简化页面结构，让 Gallery 本身成为视觉焦点 |
| 金色而非蓝色作为强调色 | 暖色调在深色背景下提供"星光"般的情感温度 |
| 灰度唤醒而非直接显示 | 赋予每张图片"被发现"的仪式感，增强浏览体验 |
| 粒子连线使用金色 | 与品牌色统一，模拟星辰之间的光之纽带 |
| Vanilla JS 零依赖 | 单文件纯静态页面，无需构建工具，开箱即用 |

## 14. 评论系统 (v6) — 二级域名部署

### 14.1 架构设计

评论系统采用**双站 + 跨域 postMessage**架构，将评论数据存储与发布能力托管到独立二级域名，避免主站膨胀：

```
主站 (shungxin.github.io)          子站 (comments.shungxin.github.io)
┌─────────────────────┐            ┌──────────────────────────────┐
│  index.html         │            │  comments.html                │
│                     │   点击     │                                │
│  卡片 💬 按钮 ─────→│ 抽屉UI     │  Giscus iframe                │
│  Lightbox 💬 ─────→│→ fetch()   │  ├─ GitHub OAuth 登录         │
│  导航 评论 ────────→│  列表      │  ├─ 真实发布到 Discussions    │
│                     │            │  └─ 实时 postMessage → 主站   │
│  postMessage 接收 ←──────────────│                                │
└─────────────────────┘            └──────────────────────────────┘
```

### 14.2 三种入口

| 入口 | 触发场景 | Thread Key |
|---|---|---|
| 顶部导航"评论"按钮 | 整站讨论 | `site` |
| 卡片左下角 💬 按钮（hover 显示） | 该图片评论 | `image-{id}` |
| Lightbox 工具栏"评论"按钮 | 当前查看图片 | `image-{id}` |

每张图片评论独立 thread，使用 `IntersectionObserver` 风格的预加载策略：卡片进入视口时即拉取该图片的评论计数并显示在 💬 按钮右上角角标。
