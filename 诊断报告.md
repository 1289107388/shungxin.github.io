# 🔍 真正的问题诊断

## 根本原因

**Supabase Edge Function 的 CORS 预检（OPTIONS）返回 500 错误**，导致浏览器拒绝发送实际请求。

### 技术细节

当浏览器发起 fetch 请求到 `https://qlhfyawbyedhqokivezn.supabase.co/functions/v1/...` 时，由于请求带自定义 header（`apikey`、`Authorization`），浏览器会**先发送 OPTIONS 预检请求**。但 Edge Function 没有正确处理这个预检请求，返回：

```
HTTP/1.1 500 Internal Server Error
Body: Internal Server Error
```

浏览器认为 CORS 失败，**直接拒绝**实际请求，导致前端显示：

```
[error] net::ERR_FAILED https://qlhfyawbyedhqokivezn.supabase.co/functions/v1/like-toggle/counts
[warn] 获取点赞数失败: Failed to fetch
[error] 点赞操作失败: 网络错误: Failed to fetch
```

### 验证

1. **从服务器直接调用 GET 请求**（PowerShell）：
   - 状态码：200 ✓
   - 响应：真实数据库数据 ✓
   - CORS 头：`Access-Control-Allow-Origin: *` ✓

2. **从浏览器发起 fetch 请求**：
   - 状态码：`net::ERR_FAILED` ✗
   - 错误：浏览器因 OPTIONS 失败而拒绝实际请求

3. **从服务器调用 OPTIONS 请求**：
   - 状态码：**500 Internal Server Error** ✗
   - 这是**真正的根本问题**

## 解决方案

需要修复 Supabase Edge Function 代码。**所有 4 个 Edge Function**（`like-toggle`、`view-count`、`comment-api`）都需要正确处理 OPTIONS 预检。

### 方案 A：修改 Edge Function（推荐）

在 Edge Function 的入口添加 OPTIONS 处理，例如（Deno 写法）：

```typescript
// supabase/functions/like-toggle/index.ts
Deno.serve(async (req) => {
  // 处理 CORS 预检
  if (req.method === 'OPTIONS') {
    return new Response(null, {
      status: 204,
      headers: {
        'Access-Control-Allow-Origin': '*',
        'Access-Control-Allow-Methods': 'GET, POST, OPTIONS',
        'Access-Control-Allow-Headers': 'apikey, Authorization, Content-Type, x-client-info',
        'Access-Control-Max-Age': '86400',
      },
    });
  }

  // ... 原有逻辑
});
```

### 方案 B：使用 supabase-js 库（应该已自动处理）

正确的 supabase-js 用法中，库会处理 CORS。但**直接 fetch** 不会。

### 方案 C：联系 Supabase 支持

如果 Edge Function 代码不可访问（已部署但无法修改），请联系 Supabase 支持。

## 需要你做的事

请到 **Supabase Dashboard** → 你的项目 → **Edge Functions**，检查以下函数的代码：

- `like-toggle/index.ts`
- `view-count/index.ts`
- `comment-api/index.ts`

确保每个函数的开头都有 `if (req.method === 'OPTIONS') return new Response(null, { status: 204, headers: {...} })` 这样的处理。

如果你的 Edge Function 是用 Supabase CLI `supabase functions new xxx` 创建的默认模板，那应该**自动处理 OPTIONS**。但根据我刚才的测试，预检请求**返回 500**，说明可能：

1. Edge Function 代码被修改过，删除了 OPTIONS 处理
2. Edge Function 部署时未启动
3. Supabase 网关配置问题

**请在 Supabase Dashboard → Logs → Edge Functions 查看是否有错误日志**。

## 临时解决方案（不推荐）

如果暂时无法修复后端，可以在前端用 `mode: 'no-cors'` 模式调用，但这样**无法读取响应数据**，所以实际上不能解决问题。
